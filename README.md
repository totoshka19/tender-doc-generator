# tender-doc-generator

Локальный инструмент автозаполнения тендерных документов.
Генерирует три DOCX-документа (Анкета участника, Заявка на участие, Предложение о цене)
на основе данных из JSON-файлов и шаблонов.

---

## Требования

- Python 3.10+
- Зависимости: `python-docx`, `PyYAML`

---

## Установка

```bash
pip install -r requirements.txt
```

---

## Структура входных файлов

| Файл | Содержимое |
|---|---|
| `data/company_profile.json` | Реквизиты компании-участника (ИНН, КПП, банк, контакт, подписант) |
| `data/tender.json` | Данные закупки (предмет, срок, позиции, НМЦ) |
| `data/calc.json` | Расчёт цены (цены без НДС, суммы позиций, итоги с НДС) |
| `data/incoming/` | Входящий DOCX от заказчика (для автоизвлечения) |

---

## Запуск

### Веб-интерфейс

```bash
uvicorn app:app --reload --port 8001
```

Открыть в браузере: http://127.0.0.1:8001

---

### CLI: Вариант 1 — с готовыми JSON-данными

```bash
python main.py --no-validate
```

Использует файлы из `data/` по умолчанию. Флаг `--no-validate` нужен для синтетических
тестовых данных с невалидным ИНН; с реальными данными запускать без него.

```bash
python main.py
```

### CLI: Вариант 2 — с автоизвлечением из входящего DOCX

```bash
python main.py --from-docx data/incoming/Запрос_ТКП_пример.docx --no-validate
```

Экстрактор прочитает DOCX, извлечёт поля (номер закупки, предмет, позиции и др.)
и сохранит результат в `data/tender_extracted.json`. Поля, которых нет в DOCX
(например, `offer_validity_days`), будут `null` — их нужно заполнить вручную,
после чего запустить генерацию с `--tender data/tender_extracted.json`.

### Все параметры CLI

```
--profile   путь к JSON профиля компании  (по умолчанию: data/company_profile.json)
--tender    путь к JSON данных тендера     (по умолчанию: data/tender.json)
--calc      путь к JSON расчёта цены       (по умолчанию: data/calc.json)
--from-docx путь к входящему DOCX          (опционально, запускает экстрактор)
--no-validate  пропустить валидацию        (для синтетических/тестовых данных)
```

## Результат

После запуска в папке `output/` появятся три файла:

```
output/
├── 01_Анкета_участника.docx
├── 02_Заявка_на_участие_в_закупке.docx
└── 03_Предложение_о_цене_договора.docx
```

---

## Добавление нового документа

1. Положить шаблон DOCX в `templates/`
2. Создать файл маппинга в `mappings/` по образцу существующих YAML
3. Запустить — новый документ будет сгенерирован автоматически

Код `engine.py` и `main.py` менять не нужно.
